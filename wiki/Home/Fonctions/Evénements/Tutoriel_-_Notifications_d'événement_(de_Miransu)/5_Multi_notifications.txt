====== 5 Multi notifications ======

Effectivement il est possible d'avoir plus d'un seul EV_USER par @Event. En fait, comme EV_USER et ses semblables sont tous des constantes, vous pouvez avoir autant d'événements définis par l'utilisateur à vérifier par script que vous le souhaitez ! Il serait tout à fait possible d'utiliser qu'un seul type d'EV_USER tout le long du jeu et de l'incrémenter à chaque nouvel événement à traiter.
En termes de code, cela se traduit ainsi pour une boucle @Event gérant tous vous événements personnalisé :
'''
@Event {
	alternative(GetEventType)
	case(EV_USER_PLAYER) { }
	case(EV_USER_PLAYER + 1) { }
	case(EV_USER_PLAYER + 2) { }
	// ... ... ...
	case(EV_USER_PLAYER + EV_USER_COUNT) { }
}
'''

Ce que vous pouvez faire avec les EV_USER et //NotifyEvent// est très vaste et seulement limité par votre imagination.
